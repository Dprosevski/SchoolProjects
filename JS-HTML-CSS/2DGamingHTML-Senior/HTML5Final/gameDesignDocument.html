Final Game Design Document

The goal of the game is to collect as many coins as possible before lava gets in the way. Speed and the starting position of the lava varies depending on the difficulty chosen before starting the game.
Each coin is a Sprite object, and each Sprite is part of the Canvas scene. Sprites are born at the top of the canvas (max y coordinate), and "randomly generated" x coordinate. A sprite dies (coin gets colleted) by chaning it's location above canvas (hidden). Each coin is created with certain set interval, and the coins are timed in that way that as the game goes on the number of dummy coins will increase. (dummy coins = coins that move or appear like they hit the lava, but they dissapear right before that happens; in other words deceiving the player).
Lava grows with constant grow rate (easy difficulty = 0.2 pixels per frame) until it reaches the point of y = addLimit px (where addLimit = 150 px from the top of canvas). It stops growing there and the only way to retract is to get the required amount of coins to be colleted.
Game stops if allowed number of coin misses is reached.
Variables that vary with difficulty: (from Easy, to Medium, to Hard)
   -grow rate of the lava (increases)
   -starting position of the lava (increases)
   -required number of coins to retract (increases)
   -retract the lava (decreases)
